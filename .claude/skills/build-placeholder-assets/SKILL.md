# Build Placeholder Assets

Goal:
Create lightweight temporary assets that let gameplay, UI, or level flow be tested before final art is ready.

Process:
1. Identify what visual feedback is needed.
2. Decide whether placeholder assets should be:
   - primitive prefab
   - simple material
   - UI blockout
   - VFX anchor
   - icon placeholder
   - editor-generated prefab
3. Use simple Unity primitives and basic colors.
4. Keep temporary assets in a dedicated placeholder folder.
5. Use `_PH` suffix for placeholder assets.
6. Add stable replacement anchors.
7. Avoid final-art assumptions.
8. Produce an artist replacement plan.
