# Unity C# Rules

## Scope
Guidance for Unity C# changes in this repository.

## Rules
- Prefer minimal diffs and focused changes.
- Do not refactor unrelated systems.
- Do not add packages without approval.
- Do not modify ProjectSettings unless explicitly required.
- Validate null safety for scene/prefab references.
- Keep serialization-safe field changes in mind.
- Include a concise verification path in final response.
